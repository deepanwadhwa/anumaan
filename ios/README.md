# Anumaan — iOS (native Swift)

Native rewrite of the navigation system, runs entirely on the phone (no laptop,
no networking). See [`../docs/swift-app-spec.md`](../docs/swift-app-spec.md) for
the full design.

## Layout
```
ios/
  AnumaanCore/        Pure-Swift navigation engine (NO UIKit) — compiles & tests via CLI
    Sources/AnumaanCore/   Models, Turn, Fusion (+HeadingCalibration), NavigationEngine
    Sources/AnumaanSelfTest/  CLI test runner (works without full Xcode)
    Tests/              XCTest suite (needs full Xcode)
  AnumaanApp/         SwiftUI app (CoreMotion sensing spike) — needs Xcode to build/run
    project.yml         xcodegen spec (source of truth for the Xcode project)
    Sources/            AnumaanApp, ContentView, SensingService
    Anumaan.xcodeproj      generated by xcodegen
```

## What works today
- **The engine is done and verified.** Dead reckoning, hard milestone gate, stop
  auto-snap, speed smoothing, no-telemetry safety, turn geometry, and the
  heading-offset calibration are all ported to Swift and pass **21/21** CLI tests.

  Run them yourself (only the Swift toolchain / Command Line Tools needed):
  ```bash
  cd ios/AnumaanCore && swift run AnumaanSelfTest
  ```

- **The app** (`AnumaanApp`) is a Phase-1 *sensing spike*: it reads CoreMotion and
  shows the live `MotionState` (moving/stopped, heading, yaw) the engine consumes.
  This proves the phone's IMU drives the engine on-device.

## Running the app on your iPhone

**You need full Xcode** (App Store, ~15 GB) — Command Line Tools alone can't build
iOS apps.

1. Install **Xcode**, then point the toolchain at it:
   ```bash
   sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
   ```
2. Open the project:
   ```bash
   open ios/AnumaanApp/Anumaan.xcodeproj
   ```
   (If you change `project.yml`, regenerate with `cd ios/AnumaanApp && xcodegen generate`.)
3. In Xcode: select the **Anumaan** target → **Signing & Capabilities** → check
   *Automatically manage signing* → **Team:** your Apple ID (Personal Team is free).
4. Plug in your iPhone, pick it as the run destination, press **▶**. Trust the
   developer profile on the phone if prompted (Settings → General → VPN & Device
   Management). Grant the **Motion** permission when asked.
5. Move/rotate the phone — `MOVING/STOPPED`, `heading`, and `yaw rate` should respond.

> Free Apple ID works; the build just needs re-deploying from Xcode every 7 days.

## Tuning note
The stop/go thresholds in `SensingService` are for `userAcceleration` (gravity
removed, ~0 at rest) and are first guesses — expect to tune them on a real drive.
The heading comes straight from CoreMotion's fused compass.
